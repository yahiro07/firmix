
# デバッグの仕方

バイナリパッチングの仕組み上、コードの移植を行う際にはこれを考慮した対応が必要になることがあります。
現在公開されているプロジェクトでは`arm-none-eabi-gcc`を使用しており、以下ではこれが前提になっています。
他のコンパイラを使う場合には別の対応が必要になる可能性があります。

以下では、コーディング上で注意が必要な点や、デバッグ時に役立つ情報をまとめます。
既存のプロジェクトを元にしてコードを書いている場合はあまり問題にならないかもしれませんが、
新しい環境でコードを書いたり、移植を行う際にコンパイラやビルドオプションが異なると予期した出力にならない可能性があります。
